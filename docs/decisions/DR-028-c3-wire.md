## Context

DR-025/026/027 landed the two C3 halves as proven, independently-verified subsystems: `BwrapSubstrate::spawn_confined` (real filesystem confinement + `--unshare-all`, folded binds, loud-OPEN degrade) and the c3bc `EgressDataplane`/`PastaHandle` (real netns inescapability + live TLS termination + real injection, proven in the WSL suite, CLOSED degrade). **Neither is wired into a live run.** The daemon spawn at `bins/rezidentd/src/runs.rs:785` builds a `SpawnPlan` and spawns the child **directly** via `tokio::process::Command::new(&plan.bin)` — it bypasses `spawn_confined` entirely and never calls `EgressDataplane::start`. So a real `rezidnt open` governed run today is neither filesystem-confined nor egress-mediated. `SandboxPolicy::from_folded_authority` and `EgressPolicy::from_folded_authority` are called nowhere in non-test code; the folded run-state carries no binds/allowlist/secret-map field, so "folded authority" has no live source yet. `spawn_confined` returns only a `SandboxedChild { backend, pid }` and detaches its OWN reaper thread (`sandbox.rs:328`), explicitly deferring "the concrete daemon-owned handle shape … to the wider run-loop" — the run loop needs the real `tokio::process::Child` to pipe stdout (`runs.rs:791`) and adopt it into the daemon reaper. And the enforce `EnforceWiring` has pasta create its OWN netns and exec a dev `probe_bin` — a self-contained test shape; in a real run the confined program is the AGENT and bwrap owns the filesystem, so pasta and bwrap must share ONE netns. This DR settles that composition. No roadmap slice exists for it: §16 stops at c3bc-enforce.

## Strongest counterargument (dissent, recorded verbatim per house style)

*"Wiring is where honest seams go to become dishonest integrations. You proved each half in a purpose-built test harness and are now gluing them under a run loop that no test drives end-to-end. The composed netns hand-off — one process seals the net namespace, the other must land inside it before the agent emits a byte — is exactly the kind of ordering bug that opens a millisecond of unmediated egress, and you have no live oracle for that window. Worse: the moment this ships, product copy starts saying 'egress is mediated,' and a single degrade-composition you got subtly wrong (sandbox open, egress silently believing it's enforcing) turns your loud-honesty story into the overclaim DR-026's own threat model warned against. Do not wire two enforcement subsystems together on the strength of two separately-green suites."*

## Counter to the counter

The dissent is right that composition is the new attack surface, and the Decision answers it structurally rather than by assertion. (1) The ordering hazard is settled by **choosing pasta-outer** — pasta seals the userspace net and execs bwrap-execs-agent *inside* the already-sealed netns, so there is no window in which the agent runs before the net route is the proxy-only route; the agent cannot emit a packet before pasta owns the namespace because pasta's exec IS the agent's ancestry. (2) The "no live oracle" objection is met by promoting the WSL escape-probe from the enforce suite to run **under the composed spawn** (criterion 2 below): the same Blocked-direct/proxy-reached falsifiable test, now driven through the real run-loop hand-off, not the test harness's self-made netns. (3) The overclaim risk is met by narrowing DR-027's module-level honesty guard to a per-primitive one: until `c3-wire` ships no copy claims mediation; once it ships, copy may claim only what a shipped run actually enforces **on the platform it enforces it** (Linux/WSL), with the three degrade states each a distinct loud logged fact so "silently believing it's enforcing" is not a reachable state. This does not weaken either DR's degrade — it defines their product.

## Decision

Carve the C3 integration slice **`c3-wire`** (Linux/WSL composition only), settling four things:

1. **One shared netns — pasta-outer, sealed-before-agent.** The composed spawn is **pasta → bwrap → agent**: pasta provides the userspace net and execs `bwrap` as its confined program; bwrap unshares everything EXCEPT net (an `unshare_all`-minus-net posture — the composed `SandboxPolicy` drops `--unshare-net` from `--unshare-all` **only when egress is active**) and execs the agent, which therefore inherits pasta's already-sealed netns rather than a fresh empty one. This matches the existing `EnforceWiring` shape (pasta execs the confined program) and guarantees the netns is sealed to the proxy-only route BEFORE the agent can emit a packet — no unmediated-egress window (DR-026 crit 3 preserved under composition; the proxy remains the sole inescapable route out). The rejected alternative — **bwrap-outer** (bwrap `--unshare-all` makes the netns, pasta attaches to the child by `/proc/PID/ns/net` after spawn) — is weaker: it has an ordering race where the agent could emit traffic in the gap before pasta attaches. We reject it for that window.
2. **The seam reshape — the daemon owns the composed child.** `spawn_confined` (or a thin composed-spawn wrapper) must hand the run loop the real **daemon-owned `tokio::process::Child`** (stdout piped, adoptable by the daemon reaper), not only a pid plus a detached waiter. This finally threads the S1 "daemon owns the process" contract that `sandbox.rs:326` deferred into the actual run loop; `runs.rs` consumes stdout and the daemon reaper reaps it, replacing the raw `tokio::Command` spawn.
3. **Folded-policy source — honestly minimal.** Per DR-025/026 the binds/allowlist/secret-map fold from the project-spec `[gates.permit]`/role layer — but that fold field does not exist yet. For this slice the honestly-scoped **first source** is the agent/gate spec config the run already carries (the same layer the badge/role/permit-hook wiring reads at `runs.rs`), passed **ONLY** through `SandboxPolicy::from_folded_authority` / `EgressPolicy::from_folded_authority`. The C6/DR-024 no-widening guard is preserved verbatim: no `SpawnPlan`-sourced or request-sourced bind/allowlist/secret — the private-field constructors stay the sole door. **If the spec carries no egress config, the allowlist is EMPTY (deny-all egress) — absent never means open.**
4. **The product of the two asymmetric degrades, in one run loop.** Sandbox degrades **loud-OPEN** (`sandbox.unavailable` + unsandboxed spawn); egress degrades **CLOSED** (`egress.unavailable` + sealed netns, no network, no injection). The load-bearing subtlety: **egress mediation is only meaningful when the sandbox netns exists** — if the sandbox degrades open there is no sealed netns for egress to be the sole route out of, so egress cannot enforce. The composed rule, each state a **distinct loud logged fact**, never a silent gap:
   - **sandbox-up + egress-up** ⇒ confined + mediated (pasta-outer, shared netns).
   - **sandbox-up + egress-down** ⇒ confined + **CLOSED**: keep the sealed netns, no network, loud `egress.unavailable`.
   - **sandbox-down (degraded open)** ⇒ unsandboxed AND a loud fact that egress is **un-enforceable** in this degraded run — no silent claim of mediation, no fake handle.

   Neither DR's degrade is weakened; this only defines their composition.

**Honesty guard (narrowed from DR-027):** once `c3-wire` ships, product/wiring copy may claim per-platform/per-primitive exactly what a shipped run actually enforces (Linux/WSL confinement + mediation); until then, no claim. No silent unconfined-or-unmediated run: the three composed degrade facts are the mitigation.

## Consequences

- **§16 delta:** adds **`c3-wire`** as the C3 integration slice **after c3bc-enforce** — the Linux/WSL composition of C3a + c3bc into the live `rezidnt open`/spawn path (pasta-outer shared netns, daemon-owned composed child, folded-from-spec first source, the three composed degrade facts). Gets its own `/oracle` → impl → `/vet` → `/debrief`; rides DR-025/026/027's ratified posture, re-designs nothing.
- **§18 risk-register delta — half-enforcement/overclaim (CARRIED, SHARPENED):** wiring is where a silent unconfined-or-unmediated run could hide. Mitigated by (a) the composed loud-degrade facts (each of the three states a distinct logged fact, no silent gap), (b) pasta-outer sealing the netns before the agent runs (no unmediated-egress window), and (c) the narrowed honesty guard — no product copy claims enforcement until this ships, and after it ships only per-platform/per-primitive what a shipped run actually enforces.
- **Deferred (flagged, not designed here):** macOS/Windows sandbox+egress backends are NOT wired — this is the Linux/WSL composition (each other-platform backend is its own DR; Windows coupled to the deferred Platform phase). The warden `/subject` for the `sandbox.*`/`egress.*`/`credential.*` facts is **still deferred** — the wiring emits its facts under placeholder subjects until the subjects are minted; flag it, do not design it.
- **No shipped test or acceptance criterion is weakened.** DR-025's loud-OPEN degrade, DR-026's crit 3/4 + CLOSED degrade, and DR-027's partitioned criteria all stand; this DR only defines their composition and adds falsifiable criteria for the seam. The one criterion that gets *stronger*: DR-026 crit 3 (inescapability) must now hold under the composed run-loop hand-off, not only the enforce suite's self-made netns.

## Acceptance criteria — `c3-wire`

1. **Spawn goes through confinement, not the raw Command.** A real `rezidnt open` governed run spawns THROUGH the composed `spawn_confined` path (pasta→bwrap→agent), confined to the folded binds — the `tokio::process::Command::new(&plan.bin)` bypass at `runs.rs:785` is gone. *(host-provable that the code path is taken + argv shape; live confinement needs the WSL box.)*
2. **Shared-netns inescapability under composition (DR-026 crit 3).** The egress dataplane is started for that run and shares the sandbox netns; an escape probe run from the RUNNING AGENT's netns is **Blocked**, and the proxy path is reached — the enforce suite's falsifiable probe, now driven through the live composed hand-off. *(WSL `#[cfg(unix)]` integration suite.)*
3. **Folded-only, no widening (C6 preserved end-to-end).** Binds/allowlist/secrets reach the sandbox/egress ONLY through `from_folded_authority`; a no-widening test fails FIRST if a `SpawnPlan`-sourced or request-sourced constructor is ever added. Empty egress spec ⇒ deny-all. *(host-provable.)*
4. **All three degrade states emit their distinct loud fact.** sandbox-up+egress-up (mediated), sandbox-up+egress-down (confined, CLOSED, `egress.unavailable`), sandbox-down (unsandboxed + loud egress-un-enforceable) — each a durable logged fact, none silently claiming enforcement it doesn't have. *(host-provable at the decision/fact level; the live sealed-netns arm needs WSL.)*
5. **The daemon owns the composed child (S1).** The composed spawn returns a daemon-owned `tokio::process::Child`; the daemon reaper reaps it and its stdout is consumed by the run loop — no detached orphan waiter. *(host-provable that the handle is returned + adopted; live reap under bwrap needs WSL.)*

Criteria 1, 3, 4 (decision arm), 5 (handle shape) are **host-provable**; criteria 2, 4 (live sealed-netns arm), and the live confinement/reap arms of 1/5 need the **WSL netns box** (`#[cfg(unix)]` integration suite) — mirroring the DR-027 host/WSL split discipline.

## Exit demo — `c3-wire`

One take, on the WSL netns box. A real `rezidnt open` governed run: the agent spawns **confined** (folded binds only, filesystem-sealed) AND **mediated** (its sole route out is the rezidnt proxy in the shared netns); an escape probe from the agent's own netns is Blocked; a clone to an allowlisted host reaches over the broker with an injected token the agent never held; an off-allowlist host is denied-and-logged; the daemon reaps the composed child and consumes its stdout. Then the three degrade facts, each shown loud: kill the connector ⇒ confined + CLOSED (`egress.unavailable`, no network); point bwrap at a missing binary ⇒ unsandboxed + a loud egress-un-enforceable fact — never a silent claim of mediation.
