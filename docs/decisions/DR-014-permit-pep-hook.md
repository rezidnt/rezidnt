# Decision Record DR-014 ‚Äî permit PEP hook binary + spawn injection (SP2 hook sub-slice)

**Date:** 2026-07-18 ¬∑ **Status:** ACCEPTED (owner) ¬∑ **Amends:** ¬ß16 (roadmap ‚Äî commits the SP2 hook sub-slice as the completing slice) / ¬ß9 (MCP surface ‚Äî records the additive socket `paths` field); no invariant text is rewritten. **Cites:** ‚Äî (no intel memo; the memo-surfaced capabilities are already scoped by DR-009). **Builds on:** [DR-013](DR-013-permit-pep-sp2-integration.md) (which deferred exactly the two questions ¬ßDecision 2/3 answer), [DR-011](DR-011-permit-pdp-config-seam.md) (empty/unresolvable ‚Üí escalate, never a synthesized allow), [DR-006](DR-006-replay-divergence-signal.md) (no consumer-less subjects); design basis [`docs/design/permit-pep-hook.md`](../design/permit-pep-hook.md), honesty constraints [`docs/design/permit-engine.md`](../design/permit-engine.md) ¬ß10.1/¬ß10.2.

## Context

DR-013 ratified the SP2 posture ‚Äî one transport-neutral `decide_permit` PDP, fail-closed to `ask`, enforcement bounded by the PEP ‚Äî but **explicitly deferred two questions to the impl slice**: (a) the concrete timeout number, and (b) where the hook lives + how a run opts into PEP enforcement. The socket-PDP half shipped (`bb7afe3`); nothing yet *asks* mid-run. The design note [`permit-pep-hook.md`](../design/permit-pep-hook.md) designs the PEP ‚Äî the claude-code `PreToolUse` hook that intercepts a live tool call, asks the daemon over the socket, and enforces the answer. It answers DR-013's two deferred questions and closes a path-scope asymmetry the auditor flagged on the `bb7afe3` /debrief. The owner chose a standalone DR-014 (note ¬ß8) over folding ¬ß2/¬ß3/¬ß5 into DR-013's deferred bucket, because this slice touches the proto wire and triggers a `/subject`.

Five surfaces need ratification, each traceable to a note section:

1. **Where the hook lives (note ¬ß2).** The PEP is a subcommand of the existing `rezidnt` CLI ‚Äî `rezidnt permit-hook` ‚Äî not a new executable. I7 (one static binary) says we do not multiply binaries; the hook is small (read stdin JSON ‚Üí one socket round-trip ‚Üí write hook output) and reuses `rezidnt-proto` types already linked into the CLI.
2. **Opt-in + injection (note ¬ß3).** Enforcement opts in through config that already exists ‚Äî no new knob. A run whose spec declares a `[gates.permit]` gate gets the PEP wired automatically at spawn via the pure, testable `SpawnPlan::for_claude_code`, which injects `REZIDNT_RUN` + `REZIDNT_SOCKET` into the scrubbed env and a `PreToolUse` hook config into the worktree settings. Run discovery is deterministic from `REZIDNT_RUN` ‚Äî never cwd-guessed. A run without a permit gate spawns exactly as today.
3. **Timeout (note ¬ß5).** DR-013 fixed the posture (fail-closed ‚Üí `ask`); DR-014 fixes the number: **250 ms default, overridable via `REZIDNT_PERMIT_TIMEOUT_MS`.** A local UDS round-trip + run-state fold is single-digit ms; 250 ms is generous headroom while keeping the hot path hot (permit-engine ¬ß10.2). On unreachable/decode-error/timeout the hook emits `ask` (escalate), never a silent proceed (restates DR-011 ¬ß3).
4. **Path-scope wire parity (note ¬ß7).** The socket `Request::RequestPermission` carries no `paths` axis, so `path-scope` degrades to escalate over the socket while the MCP path (which gained `paths` on `RequestPermissionArgs`) can deny ‚Äî a live MCP-vs-socket asymmetry the auditor flagged on `bb7afe3`. Add `paths: Option<Value>` (optional, additive) to the socket op so `path-scope` decides identically on both transports.
5. **Degradation visibility (note ¬ß6).** For `gate_explain` to honestly distinguish *mid-run-enforced* from *edge-gated* (I4), the spawn must record whether the PEP was wired. This wants a warden `/subject` ‚Äî a field on `agent.spawned` or a small `permit.enforcement.declared` subject + reducer (no consumer-less subjects, DR-006).

**Dissent (recorded verbatim per house style):** the hot-path timeout adds latency to every governed tool call against a fabric designed for ‚â§~10¬≥ events/min (permit-engine ¬ß10.2), and fail-open would keep agents moving when the daemon flaps. **Counter to the counter:** DR-013 already ratified fail-closed knowingly ‚Äî for the "may" axis a silent permit-on-failure is exactly the overclaim ¬ß10.1 forbids; the answer to latency is the decision fast-path cache (¬ß10.2), not fail-open, and 250 ms is a ceiling, not a per-call cost. DR-014 only fixes the number DR-013 left open; it does not reopen the posture.

## Decision

1. **Ratify the PEP as the `rezidnt permit-hook` CLI subcommand** ‚Äî not a new binary (I7). claude-code's `PreToolUse` config invokes it; no second artifact ships. (Note ¬ß2.)
2. **Ratify automatic opt-in keyed on `[gates.permit]`, injected via `SpawnPlan::for_claude_code`** ‚Äî `REZIDNT_RUN` + `REZIDNT_SOCKET` + the hook config; deterministic run discovery from `REZIDNT_RUN`; no new config knob. (Note ¬ß3.)
3. **Ratify the 250 ms default timeout, overridable via `REZIDNT_PERMIT_TIMEOUT_MS`.** On unreachable/decode-error/timeout the hook emits `ask`, never a silent proceed (DR-011 ¬ß3). (Note ¬ß5.)
4. **Ratify adding `paths: Option<Value>` (optional, additive) to the socket `Request::RequestPermission`** so `path-scope` decides identically over socket and MCP. (Note ¬ß7.)
5. **Commit the warden `/subject` for enforcement-mode visibility as required work ‚Äî but do NOT design the taxonomy here.** The choice between a field on `agent.spawned` and a `permit.enforcement.declared` subject + reducer is the warden's gated `/subject` session (no consumer-less subjects, DR-006). `gate_explain` then distinguishes mid-run-enforced from edge-gated (I4 degradation honesty). (Note ¬ß6.)

This is scope + posture + wire, not implementation. The build is a follow-on oracle-first slice (sequence: this DR ‚Üí warden `/subject` ‚Üí `/oracle` ‚Üí implementer ‚Üí `/vet` ‚Üí `/debrief`).

## Consequences

- **¬ß16 roadmap delta:** the SP2 **hook sub-slice** is committed as the completing slice ‚Äî the first external-boundary crossing where a live tool call is blocked, one take. Its acceptance turns the two SP2 `#[ignore]`/`unimplemented!()` stubs live (`permit_socket_decision.rs` fail-posture; `permit_pep_enforcement.rs` script-leg).
- **¬ß9 MCP-surface / proto-wire delta:** the socket `Request::RequestPermission` gains one optional additive field (`paths: Option<Value>`) ‚Äî a ¬ß9 no-drift additive change; existing senders that omit it are unaffected. `SpawnPlan::for_claude_code` gains env + hook-config injection (auditor should expect this on the pure struct).
- **Risk-register (¬ß18) deltas:** *Enforcement bounded by the PEP* (DR-008/DR-009, permit-engine ¬ß10.1) is restated **unchanged** ‚Äî the DR-013 delta stands; SP2 does not make rezidnt the sole chokepoint (C3 stays fenced behind its own DR). *Hot-path latency* (permit-engine ¬ß10.2): the 250 ms ceiling is stated, not hidden; the mitigation remains the decision fast-path cache, not fail-open.
- **Invariants touched vs untouched:** no invariant text is rewritten. **I7 upheld** ‚Äî the PEP is a CLI subcommand, no new binary. **I4** degradation is explicit ‚Äî `gate_explain` distinguishes mid-run-enforced from edge-gated, pending the ¬ß6 `/subject`. **I6 preserved** ‚Äî `ask`/`deny` never coerced to proceed; fail-closed ‚Üí `ask`. **I2** upheld ‚Äî bulky `tool_input` pins to CAS as `context_ref`, never inline over the socket. **I1** holds ‚Äî the escalate/`ask` surface is a client; the daemon renders nothing. **I3** unchanged ‚Äî run discovery is from injected `REZIDNT_RUN`, and the decision fact is still the one PDP path's output.
- **Ontology / `/subject` change IS required** (unlike DR-011/DR-012): the enforcement-visibility signal (¬ß6) is gated warden work committed but not designed here.
- **No test or acceptance criterion is weakened by this record.** In plain words: fixing the timeout at 250 ms with fail-to-`ask` keeps enforcing rather than silently permitting (a tightening over fail-open); adding `paths` to the socket op *closes* the auditor's asymmetry so the socket can now deny where it previously degraded to escalate ‚Äî a tightening, not a loosening; the additive `paths` field breaks no existing sender. The two `#[ignore]` stubs go live, replacing honest placeholders with real enforcement.
