# Decision Record DR-026 — C3b+c: mediated egress + credential brokering (full L7 MITM) — a TLS-terminating `EgressProxy` substrate that makes the daemon the sole, inescapable route out of C3a's sealed netns and brokers secrets the agent never holds, degrading CLOSED when absent

**Date:** 2026-07-20 · **Status:** ACCEPTED (owner) — ratified 2026-07-21; the owner signed off on the heaviest C3 posture commitment (a TLS-terminating daemon holding a CA private key + live brokered secrets), discharging the DR-009 fence for the folded C3b+c primitive · **Amends:** §16 (the C3 sole-chokepoint phase, whose first slice C3a shipped as [DR-025](DR-025-c3a-linux-sandbox.md), gains the folded C3b+c slice + acceptance criteria + exit demo) and §18 risk-register (secret-in-log catastrophic-failure risk NEW, CA-blast-radius risk NEW, scope-gravity largest-carried, cert-pinning functional limit) and **Appendix A dependency table** (adds `rustls`, `rcgen` — the first *linked* deps a C3 slice has added). No invariant text is rewritten. **May imply a downstream warden `/subject`** (an `egress.*`/`credential.*` subject family vs. riding the existing `permit.*` axis) — flagged here, NOT designed here. · **Builds on:** [DR-025](DR-025-c3a-linux-sandbox.md) (C3a sandbox — the containment this rides), [DR-009](DR-009-match-omnigent-scope.md) (fenced C3, requiring a design sketch + implementation DR per primitive before any build), [DR-017](DR-017-permit-macaroon-delegation-sp4b.md)/SP4b (the process-lifetime root-of-trust + the threat-model template this record mirrors), the design sketch [`docs/design/permit-egress-proxy-c3b.md`](../design/permit-egress-proxy-c3b.md) (the design basis this DR ratifies, written for the folded scope) and [`permit-sole-chokepoint-c3.md`](../design/permit-sole-chokepoint-c3.md) §4 (the decomposition the owner has now FOLDED), [permit-engine](../design/permit-engine.md) §3/§6/§10.2 (PEP as sole chokepoint + hot-path volume), and the C6 privilege-escalation lesson (DR-024 — an input that widens authority comes from folded authority, never a self-declared arg). · **Cites:** intel memo [`intel/001-omnigent-permission-governance.md`](../../intel/001-omnigent-permission-governance.md) C3 + honest-counters #7/#9/#17 (DR-002 rule 3 — our own black-box read; nothing ported).

## Context

DR-009 folded C3 (sandbox / egress proxy / credential brokering) as Omnigent's one enforcement primitive rezidnt does not match, and fenced it hard: no build per primitive without its own design sketch and implementation DR. C3a (Linux `bwrap` sandbox) shipped as DR-025 and seals the network namespace — a confined agent has **no external network** today. The parent sketch §4 decomposed the rest into C3b (egress proxy) and C3c (credential brokering) as separate slices.

The **owner has now chosen to FOLD C3b+C3c into one slice** — the full L7 chokepoint: TLS MITM (termination + body mediation) plus credential brokering, built together. The folded-scope design sketch (`permit-egress-proxy-c3b.md`) is written. This record ratifies that **folded scope** for owner sign-off. It is not a line-by-line implementation and mints no ontology. The decision stays what it has always been — an ordinary permit-verifier (`EgressScope`, a `PathConfinement`/`ForbiddenPath` sibling, `pass→proxy / fail→refuse / inconclusive→escalate`, I6, never coerced); this slice adds the **mechanism that makes that verdict the only route out** of C3a's sealed netns, and the machinery to inject a brokered secret into the upstream request after termination so the agent never holds it.

## Strongest counterargument (dissent, recorded verbatim per house style)

*"Folding C3b+C3c is the biggest single enforcement slice in the whole roadmap, and it is the most roadmap-hours away from the evidence/audit wedge §18 names as the actual differentiator. C3a at least rode the existing spawn seam at zero linked-dep cost. This slice does the opposite: it stands up a TLS-terminating proxy, a per-destination CA lifecycle, a daemon-side secret store, and a body-mediation path — and it adds real linked dependencies (`rustls`, `rcgen`) to the one-static-binary posture we have guarded since I7. Worse, it turns the daemon into a strictly larger root of trust than anything we have shipped: it now holds a signing CA that can mint trusted certs for any host the sandbox reaches AND holds live brokered tokens. That is a bigger blast radius than SP4b's MAC key, and it is Omnigent's turf — full egress interception — built at maximum cost, from behind, on ground they chose. Ship C3a's containment, keep the network sealed, and keep building the wedge until egress breadth is the thing actually blocking a user."*

## Counter to the counter

The justification is **not** enforcement breadth — it is *replayable egress and injection decisions as evidence* (one log, both axes) paired with *credential non-exposure*. Omnigent has egress interception and brokering; nobody pairs them with a replayable audit log where every allow/deny/escalate and every by-reference injection folds onto the same log as `debrief` (I3). That pairing is the differentiator, and it is only real if we build the chokepoint. The slice **rides C3a's containment** rather than reopening it (the netns is already sealed; C3b+c reintroduces exactly one route, the proxy). The dep delta is honest and bounded (below): MITM/PKI **cannot** be safely hand-rolled — the opposite of SP4b's finding that a caveat envelope over vetted blake3 was hand-rollable — so `rustls`+`rcgen` are the correct cost, not a corner cut. And the owner chose this folded scope knowingly. **The owner accepts or rejects this trade at ratification.**

## Decision (ratify scope — not a line-by-line impl)

- **Ratify the folded C3b+c scope.** One slice delivers the full L7 chokepoint: inescapable mediated egress + TLS termination + body mediation + credential brokering. C3b and C3c are no longer separately sliced.
- **Inescapable egress into C3a's sealed netns.** A userspace-net connector (recommended `pasta`/passt, exec'd like `bwrap` — rootless, no CAP_NET_ADMIN, an external binary = zero new *linked* dep) whose **sole outbound target is the rezidnt proxy**. **PIN transparent interception:** ALL outbound TCP **and** DNS redirect to the mediator — not only HTTP_PROXY-aware ports. A proxy-aware-only design is a silent hole and is REJECTED. The DR-level evaluation `pasta` vs `slirp4netns` (availability/maturity/rootless) resolves at build.
- **`EgressProxy` substrate (I4) terminates TLS.** A process-lifetime rezidnt CA mints per-destination leaf certs (`rcgen`); termination via `rustls`. The CA cert is injected **READ-ONLY** into the confined trust store (a folded bind, C3a's mechanism); the **CA private key is daemon-only, never on the fabric.**
- **The decision stays a native permit-verifier.** `EgressScope` (`pass/fail/inconclusive`, `gate_explain` names the policy + destination). The allowlist AND the which-secret-for-which-destination injection map are **FOLDED authority, never a self-declared arg** — mirroring the C6/DR-024 type-system no-widening guard C3a enforces (private-field): a run-supplied value can never widen either.
- **Credential brokering.** Secrets live daemon-side, process-lifetime, **never on the fabric** (like the SP4b root key / operator badge, DR-017). On approved egress the credential is injected into the **UPSTREAM request only**, after termination; the agent's own request carried none and it cannot exfiltrate what it never held. The injection is a durable fact **BY REFERENCE**: `credential.injected {run, dest, secret_ref, policy_ref}` — **NEVER the secret value** (I2/I3). v1 secret provenance: daemon-side/folded/never-on-fabric (richer provenance deferred, below).
- **Degrade CLOSED** (the inverse of C3a's loud-open degrade): connector, proxy, or CA unavailable ⇒ keep C3a's sealed netns (**no network**), a loud `egress.unavailable` fact, and **NO injection** — never unmediated egress, never a leaked secret.

## Consequences

- **§16 delta:** the C3 sole-chokepoint phase (first slice C3a, DR-025) gains the folded C3b+c slice + the acceptance criteria and exit demo below, so `/oracle` has concrete criteria to encode. macOS/Windows egress backends remain fenced behind the same trait, each later; no reorder of the Platform deferral is implied.
- **Appendix A dependency-table delta:** two new rows — `rustls | TLS termination (C3b+c egress MITM) | MIT/Apache/ISC | high` and `rcgen | rezidnt CA + per-destination leaf certs | MIT/Apache | high`. These are the first *linked* deps any C3 slice has added (C3a and SP4b added none); the I7 ⚠️ row above is the justification of record.
- **Risk-register (§18) deltas:**
  - *Secret-in-log catastrophic-failure risk (NEW):* a brokered secret value leaking into any fact, evidence blob, CAS ref, or trace defeats the entire primitive. Mitigated by ref-only logging (I2/I3), by the injection happening on the upstream plaintext the agent never sees, and by a property test over the **whole emitted log** (criterion 5) — the single most important test surface after inescapability.
  - *CA-blast-radius risk (NEW):* the daemon now holds a signing CA private key + live secrets — a strictly bigger root of trust than C3a (policy only) or SP4b (a MAC key). Mitigated by: the CA is process-lifetime, sandbox-scoped (injected only into confined trust stores, never the host's), and never on the fabric. Honest boundary, stated plainly: **a compromised daemon is out of scope** — same root-of-trust boundary as SP4b, but the *consequence is larger* (a compromised daemon can mint trusted certs for any host the sandbox reaches AND holds the brokered tokens). We do not claim to defend against it.
  - *Scope-gravity risk (carried, now largest):* this is the biggest single enforcement slice — TLS, CA lifecycle, secret store, injection policy, body proxying. Mitigated by binding its justification to *replayable decisions + credential non-exposure* (not breadth), by riding C3a's containment, and by the owner choosing the folded scope knowingly at ratification.
  - *Cert-pinning functional limit (carried):* an agent tool that pins certs (won't trust the injected CA) cannot connect through the MITM — inherent to TLS interception (Omnigent has it too). Such traffic is **denied-visibly, not silently mangled**; product copy notes pinned endpoints don't traverse the broker.
- **No shipped test or acceptance criterion is weakened.** This slice ADDS a substrate, a verifier, criteria, and two linked deps; it softens no existing gate, verifier, or golden. Degrade-CLOSED is not a relaxed criterion — it is the strictest possible degrade (deny + no injection), announced as a loud fact rather than passing silently.

## Acceptance-criteria sketch (what `/oracle` encodes once the slice starts)

1. A confined agent **reaches an allowlisted host through the proxy** — the request succeeds and the connection is a durable logged fact.
2. A request to a **non-allowlisted host is DENIED and logged** — a durable denial fact lands, not a silent success and not a crash.
3. **Inescapability:** direct-egress attempts — unset proxy env, a raw socket, an alternate DNS resolver — reach **nothing but the proxy**; DNS resolves *through* the mediator or is denied. No non-proxy path exists out of the netns.
4. A **brokered credential is injected on approved egress and the agent NEVER sees the secret** — it is absent from the agent's environment and from the agent's own request; only the upstream request (post-termination) carries it.
5. **The secret value NEVER appears in any fact, evidence blob, CAS ref, or trace** — a property test over the **whole emitted log** asserts only the `secret_ref` appears, never the secret bytes.
6. The **allowlist and injection map cannot be widened by any run-supplied input** (the C6 property, DR-024): a spawn arg / agent-supplied value that attempts to add an allowlisted host or a secret mapping does not widen either — both come only from folded authority.
7. **Degrade CLOSED:** connector or CA absent ⇒ **no network** + a loud `egress.unavailable` fact + **no injection** — never unmediated egress, never a leaked secret. The degrade is visible on the log.
8. **The added linked deps are `rustls` + `rcgen` only, license-clean** — the workspace gains no other new linked crate for this slice, and both licenses verify permissive (Appendix A).

## Exit demo (made concrete)

One take, headless, socket/CLI only. On a host with the connector + proxy + CA: a confined agent **git-clones/pulls from an allowlisted GitHub over the broker** — it **succeeds WITH an injected token the agent never held**, and the token appears in **NO log fact** (only `secret_ref` + `policy_ref`). A request to a **non-allowlisted host is denied-and-logged**, and that denial **replays from the log** (same recorded facts → same verdict). With the connector or CA **absent**, the same run has **no network and says so on the log** (`egress.unavailable`) and injects nothing — never opens. This is the C3b+c exit: the daemon is the sole, inescapable route out; brokered secrets are injected but never exposed and never logged; and the absence of the mediation path fails closed and announces itself.

## What this does NOT decide

- **The `egress.*`/`credential.*` taxonomy** — a new subject family vs. riding `permit.*`, with folding reducers (DR-006, no consumer-less subjects) — is a warden `/subject` session (sketch §5), not this DR.
- **macOS and Windows egress backends** — later behind the same `EgressProxy` trait; Windows gated behind the deferred native-Windows Platform phase. No reorder implied.
- **The decision fast-path cache** — a measured optimization (permit-engine §10.2), not an acceptance criterion; safe default is log-all/no-cache until latency is measured.
- **The Omnigent-baseline egress benchmark** (memo #7/#9/#17, black-box, DR-002 rule 6) — it *consumes* C3, it does not gate it.
- **Secret provenance beyond v1** — where brokered secrets come from beyond "daemon-side, folded, never on the fabric"; richer provenance (external broker, rotation) is later.
